# BCX Domain Profiles

Domain profiles define business or operational meaning above the BCX core.
They must not be confused with carrier, binding, or settlement profiles.

Future examples:

- `bcx-profile-banking`
- `bcx-profile-ai-agent`
- `bcx-profile-supply-chain`
- `bcx-profile-healthcare`

Domain crates can define mandates, approvals, tool authority, compliance
evidence, and domain-specific effect vocabulary without changing the invariant
BCX statement model.
