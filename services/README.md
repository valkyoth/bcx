# BCX Services

Services are optional applications built around the BCX protocol family. They
are not dependencies of the root crate.

Future examples:

- `bcx-cli`
- `bcx-node`
- `bcx-witness`
- `bcx-prover`
- `bcx-query`

Services may use databases, runtimes, network stacks, and operator tooling, but
protocol semantics must remain in the core/profile crates.
